## Advanced Pod Scheduling

By default, the scheduler places Pods on nodes with the most free resources. Advanced scheduling lets you override this.

### Taints & Tolerations
* **Taints** on **nodes** repel Pods (`NoSchedule`, `PreferNoSchedule`, `NoExecute`).
* **Tolerations** on **Pods** allow scheduling on tainted nodes.
* Example: `kubectl taint nodes node1 processor=gpu:NoSchedule`

[Hands-on: `1.kubernetes/3.8.deployment.toleration.yaml`]

## Node Affinity

Attracts Pods to specific nodes based on node labels.

* **`requiredDuringSchedulingIgnoredDuringExecution`**: Hard requirement.
* **`preferredDuringSchedulingIgnoredDuringExecution`**: Soft preference.

[Hands-on: `1.kubernetes/3.7.deployment.affinity.yaml`]

## StatefulSets

Used for stateful applications (databases, message brokers).

* **Stable Network Identity**: Predictable hostnames (`mysql-0`, `mysql-1`).
* **Stable Persistent Storage**: Each Pod bound to its own PV across rescheduling.
* **Ordered Deployment**: Created sequentially (0, 1, 2); deleted in reverse.
* Requires a **Headless Service** for peer-to-peer discovery.

[Hands-on: `1.kubernetes/21.statefulset.yaml`]

## DaemonSets, Jobs, and CronJobs

### DaemonSets
* Ensures every node (or a subset) runs a copy of a Pod.
* Use cases: Log collectors, monitoring agents, storage daemons.
* [Hands-on: `1.kubernetes/24.daemonset.yaml`]

### Jobs
* Runs a workload to completion (migrations, backups).
* Supports parallel and indexed completions.
* [Hands-on: `1.kubernetes/27.job.yaml`, `28.job-indexed.yaml`]

### CronJobs
* Runs Jobs on a schedule (daily backups, reports).
* [Hands-on: `1.kubernetes/30.cronjob.yaml`]

## Kubernetes Storage Architecture

Kubernetes separates storage provisioning from application deployment.

### Volumes in Pods
* **`emptyDir`**: Temporary shared storage; lost when Pod is deleted.
* **`hostPath`**: Mounts host filesystem (use with caution).
* **`nfs`**: Mounts an NFS share.
* **`configMap` / `secret`**: Mount configuration as files.

[Hands-on: `1.kubernetes/33.emptydir.yaml`, `34.nfs.yaml`]

---

## PersistentVolumes & StorageClasses

* **PersistentVolume (PV)**: Cluster-scoped storage provisioned by admin; exists independently of Pods.
* **PersistentVolumeClaim (PVC)**: Namespace-scoped request for storage; binds to a PV.
* **StorageClass**: Enables **Dynamic Provisioning** — automatically creates PVs when PVCs are requested.

[Hands-on: `1.kubernetes/35.pv.yaml`, `36.storageclass.yaml`]

## Ingress Routing

Ingress exposes HTTP/HTTPS routes from outside the cluster to Services.

* **Ingress Controller**: Required implementation (e.g., Ingress-Nginx, Traefik).
* Routes by **host**, **path**, or **headers**.
* Supports TLS termination and load balancing.

[Hands-on: `1.kubernetes/18.ingress.yaml`]

## Network Security: NetworkPolicies

By default, all Pods in a cluster can talk to each other (flat network).

* **NetworkPolicies** act as firewalls at the Pod level.
* Enforced by the CNI plugin (e.g., Calico).
* Restrict **Ingress** (incoming) and **Egress** (outgoing) traffic.
* Rules select by Pod labels, Namespace labels, or IP CIDR blocks.

[Hands-on: `1.kubernetes/37.networkpolicy.yaml`]

## Horizontal Pod Autoscaler (HPA)

Automatically scales Pod replicas based on metrics.

* **Metrics**: CPU, Memory, or custom metrics (requests per second).
* Queries the Metrics Server and adjusts Deployment/StatefulSet replica count.

[Hands-on: `1.kubernetes/48.hpa.yaml`]

## Role-Based Access Control (RBAC)

Regulates access to Kubernetes API resources.

* **ServiceAccount**: Identity for processes running inside Pods.
* **Role** / **RoleBinding**: Namespaced permissions.
* **ClusterRole** / **ClusterRoleBinding**: Cluster-wide permissions.

[Hands-on: `1.kubernetes/51.rbac.yaml`, `3.9.deployment.serviceaccount.yaml`]

## k9s: Terminal UI for Kubernetes

`k9s` makes navigating and troubleshooting clusters fast and visual.

| Key | Action |
|-----|--------|
| `:pods` | View all Pods |

| `:svc` | View Services |
| `d` | Describe resource |
| `l` | View logs |
| `e` | Edit resource |
| `ctrl+d` | Delete resource |
